# 氧化锆电导率数据清洗实验报告

## 1. 实验目的
本实验旨在针对氧化锆（Zirconia）基材料电导率的原始科研数据进行自动化清洗与标准化处理。目标是构建高质量、结构化的数据库，解决原始数据中格式混乱、描述非结构化及单位不统一等问题，为后续的物理信息神经网络（PIML）模型训练提供可靠的数据支撑。

## 2. 实验环境
* **开发语言**：Python 3.x
* **数据库**：MySQL 8.0+
* **核心库**：Pandas, SQLAlchemy, PyMySQL, LangChain (OpenAI API), python-dotenv
* **数据源**：`data_20251205.xlsx` (包含原始文献挖掘数据)

## 3. 数据架构设计
为了有效存储复杂材料属性，实验设计了符合第三范式（3NF）的关系型数据库模式：

* **raw_conductivity_samples**：原始数据缓冲表。所有字段均设为 `TEXT` 类型，确保 Excel 中的任何非标准格式都能无损导入。
* **material_samples**：清洗后的核心主表。存储 `sample_id`、参考文献及标准化的电导率、工作温度等物理量。
* **sample_dopants**：掺杂元素明细表。解决"一行多值"问题，将 `Y/Sc/Dy` 等混合掺杂拆分为多行存储，包含离子半径、价态及摩尔分数。
* **sintering_steps**：烧结步骤表。用于存储多阶段热处理工艺（如 "1400℃, 2h -> 1500℃, 4h"）。
* **sample_crystal_phases**：晶相构成表。关联样本与 `crystal_structure_dict` 字典表，明确材料的晶体结构（如立方相、单斜相）。

## 4. 实验流程与实现

### 4.1 数据加载 (Ingestion)
使用脚本 `load_raw_data_to_mysql.py` 执行初步加载。
* **策略**：采用 `dtype=str` 强制读取，避免 Pandas 自动推断类型导致的精度丢失或格式错误。
* **处理**：将 Excel 中的空单元格统一转换为空字符串，确保数据完整入库。

### 4.2 自动化智能翻译 (LLM-based Translation)
利用大语言模型（LLM）对非结构化的中文描述进行标准化翻译：
* **原材料标准化**：运行 `translate_column_material_source_and_purity.py`，使用 LangChain 调用 OpenAI 接口，将复杂的中文供应商及纯度描述（如"商品化材料(Toyo Soda)，>99%"）重写为标准的学术英语句式。
* **工艺描述标准化**：运行 `translate_column_synthesis_method_and_processing_route.py`。
    * **优化策略**：采用"提取唯一值 -> 批量翻译 -> 映射回库"的策略，大幅减少 Token 消耗。
    * **并发处理**：使用 `chain.batch` 实现并发请求，提高处理效率。

### 4.3 结构化清洗与转换 (ETL)
核心清洗逻辑由 SQL 脚本 `etl.sql` 完成，主要包含以下操作：
1.  **数值清洗**：利用 `REGEXP_SUBSTR` 正则表达式从混合文本中提取温度和时间数值。
2.  **单位统一度量**：编写 `CASE WHEN` 逻辑，将不同单位的时间（`h`, `min`）统一换算为分钟。
3.  **科学计数法修正**：针对电导率字段，将文本中的 `×10`、`*10` 替换为标准的 `E` 记号，并转换为双精度浮点数 (`DOUBLE`)。
4.  **维度拆分 (Normalization)**：
    * 利用 SQL `CROSS JOIN` 生成数字序列，配合 `SUBSTRING_INDEX` 函数，将单行中的复合掺杂信息（`Y/Sc/Fe`）拆解为多行独立的记录插入 `sample_dopants` 表。

## 5. 实验结果校验
通过执行 `test_clean_data.sql` 对清洗结果进行了抽样验证：
* **数值转换**：成功识别并转换了如 `1.2×10^-2` 等非标准科学计数法格式。
* **空值清洗**：原始数据中的占位符 `/` 已被正确转换为数据库的 `NULL` 值。
* **结构化验证**：样本 `sample_id=193` 的多步烧结数据成功拆分为 `sintering_steps` 表中的多条有序记录。

## 6. 实验结论
本实验成功实现了一套"Python控制 + LLM语义增强 + SQL逻辑处理"的混合式数据清洗管线。该方案有效解决了材料科学数据中存在的非结构化文本多、多组分耦合度高、格式不统一等难题，最终生成的结构化数据库字段清晰、类型规范，完全满足后续机器学习建模的需求。
